Shared gallery progress (Supabase)
===================================

1. Create a free project at https://supabase.com

2. Open SQL Editor, paste and run: supabase/schema.sql

3. Project Settings → API: copy "Project URL" and "anon public" key

4. Copy supabase-config.example.js to supabase-config.js

5. Set MADHU_SUPABASE_URL and MADHU_SUPABASE_ANON_KEY in supabase-config.js

6. Deploy your site with both index.html and supabase-config.js (same folder as script.js)

Vercel (recommended — no secrets in git):
- Project → Settings → Environment Variables → add:
  - MADHU_SUPABASE_URL = your Supabase project URL
  - MADHU_SUPABASE_ANON_KEY = anon public key
- Redeploy. Build runs `npm run build`, which writes supabase-config.js on the server from those vars.
- `supabase-config.js` stays gitignored locally; Vercel generates it during deploy.

Without supabase-config.js filled in (locally) or env vars set (Vercel), the site still works using only localStorage on each device.

Reset testing data:
- Add ?resetGallery=1 to the URL (clears this browser + uploads empty state if Supabase is configured)

Security note: anon RLS policies allow anyone with your URL + anon key to read/write this one table.
